# 《高校学习智能体 AIGC 应用》Agent 特性功能清单

**文档说明**：本清单基于《高校学习智能体AIGC应用开发方案》梳理，旨在明确界定系统所具备的自主感知、决策、执行与闭环优化能力，作为论证产品为“Agent 应用”而非“纯交互界面”的核心依据，可直接用于架构评审与研发验收。

## 核心功能清单对比表

| 功能名称 | Agent 特征描述 (感知/决策/执行/闭环) | 技术实现要点 (触发条件/算法模型/数据流/反馈机制) | 与纯交互界面的本质差异 |
| :--- | :--- | :--- | :--- |
| **1. 复杂意图自主分解与多路路由分发 (Supervisor Routing)** | **感知与决策**：能够理解复合型指令（如“查资料并排个复习表”），自主将大任务分解为原子任务，并决策将其分发给不同的领域专家Agent（答疑、编程、规划等）。 | **触发条件**：用户输入长文本或复合需求。<br>**算法/模型**：LLM 结构化输出 (JSON Schema) + LangGraph 状态机路由。<br>**数据流**：用户 Prompt → Supervisor 解析 → 生成并行子任务流 → 唤起对应子 Agent。<br>**反馈机制**：子 Agent 返回状态给 Supervisor，若失败则触发重试调度。 | **纯交互界面**只能作为一个单体大模型按顺序生成长文本；**Agent** 相当于“项目经理”，能拆解任务并指挥多个“下属”协同并行工作。 |
| **2. 代码沙箱自主运行与错误自愈 (Self-Correction Sandbox)** | **执行与闭环优化**：在接收到编程问题时，不仅生成代码，还会在安全的 Python 沙箱中自主运行代码。若捕获到报错（Error Log），会自动阅读日志、修改代码并再次运行，直到跑通。 | **触发条件**：触发代码调试或计算需求。<br>**算法/模型**：DeepSeek-Coder 等代码模型 + Docker 隔离沙盒 API。<br>**数据流**：生成代码 → 沙盒执行 → 捕获 Stdout/Stderr → LLM 分析修复 → 重新执行。<br>**反馈机制**：Exit Code 0 为成功标志，若连续 3 次非 0 则抛出异常给用户。 | **纯交互界面**仅输出一段代码文本，需用户自行复制、运行和排错；**Agent** 具备真实的机器执行权限，能完成“写-跑-修”的全自动闭环。 |
| **3. 强溯源与事实核查的知识组装 (Fact-Checked RAG)** | **感知与决策**：面对学术提问，自主从向量库检索资料，评估检索到的上下文是否足以回答问题。若不足，主动拒绝回答（防幻觉）；若充足，强制附带精确页码与出处。 | **触发条件**：课程知识问答类意图。<br>**算法/模型**：BGE-M3 向量检索 + Reranker 重排模型 + LLM 事实核查提示词。<br>**数据流**：Query → 向量库召回 Top K → Rerank → 上下文质量评估 → 生成带引用标识的回答。<br>**反馈机制**：基于事实核查阈值拦截不合格输出，低于阈值则返回“资料不足”。 | **纯交互界面**容易产生大模型“幻觉”胡编乱造；**Agent** 拥有“知识审计”能力，对自己的输出进行事实比对并强制加注来源。 |
| **4. 跨系统生态整合与主动触达 (Cross-System Workflow)** | **感知与执行**：不仅在聊天框内对话，还能通过 API 插件读取教务系统数据（如课表、成绩），并根据状态机设定，主动通过企业微信/飞书向学生推送学习提醒。 | **触发条件**：定时任务触发 (Cron) 或特定规划请求触发。<br>**算法/模型**：Function Calling / Tool Use + RESTful API 交互。<br>**数据流**：Agent 触发工具 → 教务 API 返回 JSON → 生成计划表 → 调用 IM Webhook 发送消息。<br>**反馈机制**：通过 API HTTP 状态码确认执行是否成功，支持断点续传。 | **纯交互界面**是被动的“一问一答”框，无法感知外部系统状态；**Agent** 具备主动出击的能力，能读写外部数据库并跨软件（IM）触发动作。 |
| **5. “苏格拉底式”合规拦截与引导策略 (Ethical Guardrails)** | **感知与决策**：当感知到学生试图“作弊”（如要求直接写期末作业）时，自主挂起常规生成流程，切换到“合规导师模式”，拒绝直接给答案，改为分步骤的启发式提问。 | **触发条件**：命中作业/考试意图且检测到直接索要答案的倾向。<br>**算法/模型**：意图分类器 (Intent Classifier) + 护栏策略 (Guardrails Prompting)。<br>**数据流**：用户输入 → 审查 Agent 拦截 → 阻断原有链路 → 载入启发式提示词模板 → 输出引导问题。<br>**反馈机制**：根据学生后续的回复，动态调整引导深度。 | **纯交互界面**是绝对服从的工具，容易沦为作弊利器；**Agent** 具备教学伦理和自主拦截的决策能力，引导而非包办。 |
| **6. 长周期任务的状态记忆与断点续跑 (Stateful Long-Task)** | **感知与执行**：对于撰写长篇论文大纲、长期复习计划等任务，Agent 能将进度和中间产物持久化保存到记忆中。即使用户关闭页面，下次回来也能无缝接续上一个节点继续执行。 | **触发条件**：长周期工作流或会话中断。<br>**算法/模型**：LangGraph 状态机 (State) + 长期记忆库 (Postgres/Redis)。<br>**数据流**：当前节点产物 → 序列化写入 DB → 会话恢复 → 反序列化载入 State → 恢复断点。<br>**反馈机制**：每次流转更新 State 版本号，支持时光机回溯（Time-travel）。 | **纯交互界面**通常是无状态的单次会话（或仅有简单的 Token 历史）；**Agent** 维护了复杂的工作流状态树，支持异步、持久化的长任务推进。 |

---

**评审总结**：
从上述清单可以明确得出结论：本方案设计的系统远超“大模型套壳 UI”的范畴。其核心竞争力在于通过 **Supervisor 分发、工具调用（Tool Calling）、代码沙盒执行、状态机维护（State Machine）** 等工程化手段，赋予了 AI **主动感知环境、使用工具改变环境、并根据环境反馈进行自我修正**的能力，是标准的 Agentic Workflow（智能体工作流）架构。
